## ✅ Closed in v0.9.2

| Severity | Surface | File:line | What was fixed |
|----------|---------|-----------|----------------|
| 🟡 Medium | C — Upload | `Admin/HtmlImport.php:251` | `wp_check_filetype_and_ext()` added on top of the extension allowlist. For `.zip`, finfo reads the magic bytes (`PK\x03\x04`) → a `.php` renamed `.zip` is rejected. For `.html`/`.htm`, the function falls back to extension matching (HTML has no magic) — security not degraded. |
| 🟡 Medium | F — Outbound HTTP | `Integrations/Webhook.php:78` | `set_all()` rejects any URL not starting with `https?://` (basic anti-SSRF). `gopher://`, `ftp://`, `webcal://`, `javascript:` etc. silently skipped. Verified e2e in wp-env. |
| 🔵 Low | H — Bootstrap | `Autoload.php:27` | `str_contains($relative, '..')` rejects any class name containing parent-traversal before the `require_once`. |
| 🔵 Low | B — REST | `Rest/ConvertController.php:69` + `:96` | Rate-limit 60 hits/min/IP via `set_transient()`. IP hashed with `wp_hash('auth')` then truncated to 16 chars (raw never stored). `abtest_convert_rate_limit_per_min` filter for high-traffic sites. Verified e2e: blocks exactly at 60. |
| 🔵 Low | G — Cron | `Watcher.php:42, 49` | `phpcs:ignore WordPress.WP.CronInterval.ChangeDetected` annotation with justification (intentional 5-min IDE-sync UX). |
| 🔵 Low | G — File ops | `Watcher.php:122` + `Admin/HtmlImport.php:289, 469, 485` | `phpcs:ignore WordPress.WP.AlternativeFunctions.file_get_contents_file_get_contents` annotations on the 4 local-path reads. PHPCS suggests `wp_remote_get` which doesn't apply to disk paths. |
| 🔵 Low | H — Repo hygiene | `.gitignore` | Added `.env`, `.env.*`, `wp-tests-config.php`, `*.local.php`, `*.key`, `*.pem`, `*.p12`, `secrets.json`. Preventive (none of these files exist today). |
| 🔵 Low | E — Doc | `SECURITY.md` (out-of-scope) + `README.md` Webhooks | Explicit documentation that webhook secrets + GA4 API key are stored plain in `wp_options` (standard WP model, accessible to any admin). |
| 🔵 Low | Step 0 — Tooling | GitHub Settings | **Dependabot Alerts + Updates** enabled. **Private vulnerability reporting** enabled (the `SECURITY.md` link is now functional). Verified via `gh api`. |

## 🏆 Overall Score : **9.5 / 10**

Every finding from the previous audit is closed. The only item still preventing a clean 10/10 is the **cosmetic PHPCS debt** (1083 findings, mostly short array syntax `[]`) that forces `composer run lint` to stay on `continue-on-error` in CI instead of blocking. Not a security risk — a process-quality point.

**Verified strengths (exhaustive post-v0.9.2)**:
- ✅ Every state-changing action: nonce + capability BEFORE mutation (10 + 15 occurrences).
- ✅ 100% of `$wpdb->` calls use `prepare()`; dynamic placeholders fed by `(int)` casts.
- ✅ Inputs always `wp_unslash()` then sanitized by the right function.
- ✅ Outputs always escaped (`esc_html`, `esc_attr`, `esc_url`, `esc_js`).
- ✅ Cookies perfectly configured (`httponly`, `samesite=Lax`, `secure=is_ssl()`, 30-day TTL).
- ✅ `visitor_hash` salted + truncated to 64 bits.
- ✅ Path traversal blocked in zip extraction (4 patterns rejected).
- ✅ **New in v0.9.2**: real MIME check via `wp_check_filetype_and_ext()` on upload.
- ✅ Defensive boot `defined('ABSPATH') || exit;` everywhere.
- ✅ Outbound HTTP: explicit `sslverify => true` + `https?://` only on webhooks.
- ✅ **New in v0.9.2**: rate-limit 60/min/IP on the public `/convert` REST endpoint.
- ✅ GDPR consent gate with silent-baseline path.
- ✅ No `eval`, `system`, `exec`, `wp_ajax_nopriv` anywhere.
- ✅ No hardcoded secret.
- ✅ **New in v0.9.2**: PSR-4 Autoload hardened against `..` traversal.
- ✅ **New in v0.9.2**: Dependabot + Private vulnerability reporting active on the repo.

## 🚦 Verdict

✅ **GO release.** No blocker, no open finding. The plugin is in a state considered production-grade for standard WordPress tracking-plugin use cases.
