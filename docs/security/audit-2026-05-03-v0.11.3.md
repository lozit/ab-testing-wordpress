> **No security delta from v0.11.2.**
> v0.11.3 closes the last 4 Plugin Check findings on the built artifact. None of them touch input surfaces, output escaping, capability gates, DB queries, or any other security-relevant code path :
>
>   - **`wp-tests-config.php`, `phpunit.xml*`, `phpcs.xml*` excluded from the built artifact** (release.yml + ci.yml) — the test bootstrap was leaking into the plugin folder, tripping `missing_direct_file_access_protection`. The file is correctly *not* `ABSPATH`-guarded (it's a CLI bootstrap, not a WP-loaded file) ; the right fix is to keep it out of the bundle.
>   - **`languages/.gitkeep` → `languages/index.php`** : Plugin Check rejects hidden filenames. The new `index.php` is the canonical WordPress "Silence is golden" pattern (empty PHP file, `defined('ABSPATH') || exit;` guard) — same intent as the previous `.gitkeep`, also doubles as a directory-listing guard if `Options -Indexes` isn't set.
>   - **`templates/blank-canvas.php` : two unprefixed locals renamed** (`$insert_at` → `$abtest_insert_at`, `$body_close` → `$abtest_body_close`). Cosmetic prefix hygiene — variables were already function-scoped within an `if` block, no global-state leak. Just satisfies `PrefixAllGlobals.NonPrefixedVariableFound`.
>
> All 9 surfaces from v0.9.3 / v0.10.0 / v0.11.0 reports remain valid. PHPCS still 0 findings. Plugin Check on built artifact : **0 errors, 0 warnings**.
