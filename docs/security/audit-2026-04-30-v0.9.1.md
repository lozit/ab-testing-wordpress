## 🔴 Critical

**No Critical vulnerability identified.**

- **SQL Injections**: every `$wpdb->` (Stats.php, Tracker.php, Experiment.php, Schema.php, Plugin.php migrations, Watcher) uses `$wpdb->prepare()` with typed placeholders (`%d`, `%s`).
- **Dynamic File Inclusion**: 2 dynamic `require_once` (Autoload PSR-4 + `vendor/autoload.php`), both plugin-controlled. No `$_GET` / `$_POST` / `$_SERVER` reaches an `include` / `require`.
- **Arbitrary Code Execution**: `eval` / `system` / `exec` / `passthru` / `shell_exec` / `popen` / `proc_open` / `assert` / `call_user_func($_GET[...])` → **zero hits** across the plugin.
- **Hardcoded secrets in repo**: `grep API_KEY|SECRET|PASSWORD|TOKEN` on `git ls-files` (excluding vendor/composer.lock/docs/CLAUDE.md/.claude) → zero hits.

## 🟠 High

**No High vulnerability identified.**

- **XSS**: every HTML output goes through `esc_html()`, `esc_attr()`, `esc_url()`, `esc_js()`. The 4 `onclick=` handlers found (`ExperimentsList.php:257, 268, 273` + `Settings.php:140`) are either static or escaped via `esc_js()`.
- **CSRF**: 10 `check_admin_referer()` + 15 `current_user_can()` occurrences. Every `admin_post_*` handler (9 hooks) has both BEFORE any mutation. Verified across all: `handle_save:172-175`, `handle_status_change:344-347`, `handle_resume:419-422`, `handle_replace_running:470-473`, `handle_delete:523-526`, `HtmlImport::handle_upload:208-211`, `HtmlImport::handle_scan_now:172-175`, `Settings::handle_save`, `Settings::handle_test_webhook`.
- **Access Control**: `ConvertController::permission_callback = '__return_true'` is intentional (public tracking endpoint, documented); `StatsController::permission_callback = [check_permission]` requires `manage_options`. No `wp_ajax_nopriv_*`. Every file has `defined('ABSPATH') || exit;`.

## 🟡 Medium

### [MEDIUM] File upload: no real MIME check
- **File**: `includes/Admin/HtmlImport.php`
- **Line**: 240-251
- **Surface**: C — File upload + zip extraction
- **Problematic code**:
  ```php
  $ext  = strtolower( (string) pathinfo( $name, PATHINFO_EXTENSION ) );
  if ( ! in_array( $ext, self::ALLOWED_EXTS, true ) ) {
      self::redirect_error( ... );
  }
  ```
- **Risk**: only the filename extension is checked. A `manage_options` attacker can upload a `.php` renamed `.html` — an executable PHP file lands under `wp-content/uploads/abtest-templates/{slug}/`. Mitigated by: (a) admin capability required, (b) the WP default `.htaccess` blocks PHP execution under `/wp-content/uploads/`, (c) sandboxed iframe preview. But the `.htaccess` is NOT always in place (Nginx, some hosts).
- **Fix**:
  ```php
  $checked = wp_check_filetype_and_ext( $tmp_name, $name, [
      'html' => 'text/html',
      'htm'  => 'text/html',
      'zip'  => 'application/zip',
  ] );
  if ( ! $checked['ext'] || ! $checked['type'] ) {
      self::redirect_error( __( 'File MIME type does not match its extension.', 'ab-testing-wordpress' ) );
  }
  ```

### [MEDIUM] Webhook URL accepts non-HTTP schemes
- **File**: `includes/Integrations/Webhook.php`
- **Line**: 73 (`set_all`)
- **Surface**: F — Outbound HTTP
- **Problematic code**:
  ```php
  $url = isset( $entry['url'] ) ? esc_url_raw( (string) $entry['url'] ) : '';
  ```
- **Risk**: `esc_url_raw()` accepts by default `http`, `https`, `ftp`, `ftps`, `gopher`, `mailto`, `sms`, `webcal`, `xmpp`, … A compromised admin (or malicious insider) could set `gopher://internal-service:11211/...` to pivot toward internal services (memcached, redis, xmlrpc) → basic SSRF. Mitigated by admin capability.
- **Fix**: enforce http(s) only.
  ```php
  $url = isset( $entry['url'] ) ? esc_url_raw( (string) $entry['url'] ) : '';
  if ( '' !== $url && ! preg_match( '#^https?://#i', $url ) ) {
      continue; // Reject non-HTTP(S) schemes (basic SSRF guard).
  }
  ```

## 🔵 Low

### [LOW] PSR-4 Autoload doesn't refuse `..` in class names
- **File**: `includes/Autoload.php`
- **Line**: 26-30
- **Surface**: H — Direct file access + bootstrap
- **Problematic code**:
  ```php
  $relative = substr( $class, strlen( $prefix ) );
  $path     = ABTEST_PLUGIN_DIR . 'includes/' . str_replace( '\\', '/', $relative ) . '.php';
  if ( is_readable( $path ) ) {
      require_once $path;
  }
  ```
- **Risk**: if some other code on the site does `class_exists($user_controlled)` with the `Abtest\` namespace, `str_replace('\\', '/')` would turn `Abtest\..\..\evil` into a traversal path. Current risk: nil (the plugin doesn't expose any `class_exists($user_input)`). 1-line hardening.
- **Fix**:
  ```php
  if ( str_contains( $relative, '..' ) ) {
      return;
  }
  ```

### [LOW] REST `/abtest/v1/convert` has no IP rate-limit
- **File**: `includes/Rest/ConvertController.php`
- **Line**: 33-50
- **Surface**: B — REST endpoints
- **Problematic code**: public endpoint, dedup by `visitor_hash` prevents one visitor from double-counting, BUT an attacker with N IPs can flood it to bias the stats of a variant.
- **Risk**: stats pollution / fake statistical significance. Not a theft, not an exec.
- **Fix**: per-IP transient bucket.
  ```php
  $ip_hash = wp_hash( (string) ( $_SERVER['REMOTE_ADDR'] ?? '' ) );
  $key     = 'abtest_rate_' . $ip_hash;
  $count   = (int) get_transient( $key );
  if ( $count >= 60 ) { // 60 hits / minute
      return new \WP_REST_Response( [ 'logged' => false, 'reason' => 'rate_limited' ], 429 );
  }
  set_transient( $key, $count + 1, MINUTE_IN_SECONDS );
  ```

### [LOW] Webhook HMAC secret stored plain in `wp_options`
- **File**: `includes/Integrations/Webhook.php`
- **Line**: 80 (`Webhook::set_all`)
- **Surface**: E — Cookies + hashing (extension: secrets)
- **Problematic code**: `$clean[] = [ ..., 'secret' => (string) $entry['secret'], ... ]` then `update_option(self::OPTION_KEY, $clean)`.
- **Risk**: any admin with read access to options (i.e. any admin) can read the HMAC secret. Standard WP storage, not encryptable without external key management. Bounded by the WP threat model (admin = trusted).
- **Fix**: document in README that "secret stored in DB, accessible to any admin" — that's the WP model. More robust solutions (at-rest encryption via an external vault) are out of plugin scope.

### [LOW] PHPCS warning `WordPress.WP.CronInterval` on Watcher (5 min < 15 min threshold)
- **File**: `includes/Watcher.php`
- **Line**: 36
- **Surface**: G — WP-Cron + filesystem
- **Risk**: no security risk — design choice (fast IDE sync). PHPCS surfaces it because too-frequent cron can weigh on high-traffic sites. The hook only does light, bounded DB ops.
- **Fix**: explicit annotation.
  ```php
  // phpcs:ignore WordPress.WP.CronInterval.ChangeDetected -- intentional 5-min for IDE-sync UX
  ```

### [LOW] PHPCS warnings `file_get_contents() is discouraged` (false positives)
- **Files**: `includes/Watcher.php:120`, `includes/Admin/HtmlImport.php:263, 443, 459`
- **Surface**: G — WP-Cron + filesystem
- **Risk**: none — PHPCS suggests `wp_remote_get()` which is for remote URLs; here these are local disk paths (uploaded file, files under uploads/abtest-templates/).
- **Fix**: annotate with `// phpcs:ignore WordPress.WP.AlternativeFunctions.file_get_contents_file_get_contents -- local file path, wp_remote_get does not apply`.

### [LOW] `.gitignore` doesn't explicitly exclude credential files
- **File**: `.gitignore`
- **Surface**: H — Direct file access + bootstrap
- **Risk**: no `.env` / `wp-tests-config.php` (with creds) / `*.local.php` exists today, but the preventive net is missing. If a dev creates one without thinking, it could be committed.
- **Fix**: append to `.gitignore`:
  ```
  .env
  .env.*
  wp-tests-config.php
  *.local.php
  *.key
  *.pem
  ```

### [LOW] GitHub Dependabot Alerts disabled
- **File**: repo settings (out of code)
- **Surface**: Step 0 — automated tooling
- **Risk**: `composer audit` runs manually and in CI on every push. Composer CVEs evolve daily → longer detection window without continuous Dependabot. Also: npm CVEs (workflow scripts) are not monitored.
- **Fix**: enable in **Settings → Code security → Dependabot alerts** (free). Same with **Private vulnerability reporting** in the same section to enable the Security disclosure tab.

**Verified strengths**:
- Every state-changing action: nonce + capability BEFORE mutation (10 + 15 occurrences).
- 100% of `$wpdb->` calls use `prepare()`; dynamic placeholders fed by `(int)` casts.
