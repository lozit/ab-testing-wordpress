>   - **Text domain replaced** in every `__()` / `_e()` call across `includes/` (~ 290 string occurrences). No new input surface тАФ text domain is not user-controlled.
>   - **Main plugin file renamed** `ab-testing-wordpress.php` тЖТ `uplift-ab-testing.php` via `git mv` (history preserved). `register_activation_hook(__FILE__, ...)` etc. auto-resolve via `__FILE__` so behavior unchanged.
>   - **Composer / npm package names** updated. `composer.lock` regenerated only as a side-effect of the name change ; no new dependency, no version bump on existing deps.
>   - **CI + release workflows** : build path `build/ab-testing-wordpress` тЖТ `build/uplift-ab-testing`, distribution zip filename updated. No change to the rsync exclusion list (still excludes the same dev-only files added in v0.11.{2,3}).
>
> **Internal names preserved** (security-relevant : a rename here would invalidate the audit baseline) :
>   - PHP namespace `Abtest\` тАФ unchanged
>   - Hook prefixes `abtest_*` тАФ unchanged
>   - Cookie names `abtest_*` (inc. `httponly`, `samesite=Lax`, `secure`, 30-day max TTL) тАФ unchanged
>   - REST namespace `abtest/v1` тАФ unchanged (would break existing API integrations otherwise)
>   - Custom table `wp_abtest_events` тАФ unchanged (would break every existing install)
>   - Option keys `abtest_settings`, `abtest_db_version` тАФ unchanged
>   - `visitor_hash` salting + 64-bit truncation (v0.8.2) тАФ unchanged
>   - HMAC SHA-256 webhook signing тАФ unchanged
>   - Consent gate (v0.8.0 `Consent::is_blocked()`) тАФ unchanged
